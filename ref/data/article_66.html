В Системе существует механизм переноса бизнес-логики из одной инсталяции (например, тестовой) Системы в другую (например, продуктивной). Процесс переноса состоит из нескольких этапов:

  1. Настройка индекса для использования в качестве первичного ключа реконсиляции.
  2. Экспорта необходимых данных.
  3. Импорт данных, выгруженных на первом шаге.

Подробнее о том, как настроить индекс для использования в качестве первичного ключа реконсиляции, см. в разделе Добавление и редактирование индексов.

Для того, чтобы выполнить экспорт данных:

  1. Перейдите к пункту «Система управления» панели навигации, расположенной в левой части экрана. Выберите пункт «Настройка– Загрузка данных - Экспорт данных».
  2. Нажмите кнопку +Добавить запись.
  3. В открывшемся окне заполните поле Отображаемое имя
  4. Нажмите кнопку Создать.
  5. Нажмите кнопку +Добавить запись в поле «Entry List».
  6. В открывшемся окне заполните следующие поля:
    • Таблица - объект системы из которого будут выгружены данные).
    • Запрос - код на языке SQL, уточняющий какие именно данные из указанного объекта необходимо экспортировать, например, для выгрузки всех записей со значением поля ORGUNIT равным 29 или 30: ORGUNIT IN (29, 30)
  1. Нажмите кнопку Создать, затем Готово.
  2. Повторите шаги 5-7 необходимое число раз, указав все данные для выгрузки из системы.
  3. Нажмите кнопку «Выгрузить данные».
  4. В поле «Export Result Id» нажмите кнопку «Загрузить».

Таким образом вы получите файл в формате .xlsx, в котором содержатся необходимые для переноса в другую инсталляцию Системы данные.

  1. Перейдите к пункту «Система управления» панели навигации, расположенной в левой части экрана. Выберите пункт «Настройки – Перенос данных - Импорт данных».
  2. Нажмите кнопку +Добавить запись.
  3. В поле Import File Id укажите путь к файлу, в котором хранятся данные для импорта.
  4. Нажмите кнопку Создать, затем Загрузить данные из файла.
  5. Проанализируйте поле Лог загрузки: в случае наличия ошибок внесите изменения в файле с загружаемыми данными, проверьте настройку индексов в обоих инсталляциях Системы и повторите шаги 1-4.
