В системе для каждому объекту можно добавить Экран, определяющий какой вид отображения будут иметь данные, хранящиеся в объекте.

Для каждого объекта Системы, за исключением тех, у которых Тип источника данных==Запрос, автоматически при создании генерируется два типа отображение:

  • DETAILS (форма просмотра одной записи).
  • LIST (форма просмотра списка записей).

Добавления нового отображения уже существующей таблицы:

  1. Перейдите к пункту «Настройка» панели навигации, расположенной в левой части экрана. Выберите пункт «Описание таблиц».
  2. В открывшемся окне перейдите в секцию Настройки отображения и нажмите кнопку +Добавить запись.

                                                            

     3. В открывшемся окне заполните необходимые поля (Обязательные поля подсвечиваются красным цветом)                                                                                                                                                                                                                                               

  • Наименование - отображаемое имя экрана
  • Системное имя - имя, которое будет присвоено отображению в системе
  • Тип отображения - для отображений, отличных от SIMPLE_LIST, LIST, необходимо будет заполнить дополнительные поля, подробнее в Создание и редактирование отчетов
  • Запрос для выборки данных - условие, которому должны удовлетворять все записи таблицы, попадающие в отображение
  • Шаблон заголовка - отображаемое имя экрана
  • Способ фильтрации (Форма поиска, Фильтр) - форма поиска используется для фильтрации полей по полям поиска, указанным в секции Поля поиска. Для добавления поля поиска
    • Если отображение не создано, перейдите в секцию Общая информации, нажмите кнопку создать
    • Перейдите в секцию Поля Поиска
    • Нажмите кнопку добавить запись
    • В открывшемся окне заполните необходимые поля (аналогично созданию атрибутов объекта)
    • Нажмите кнопку создать
  • Экран просмотра одной записи - из выбранного списка выберите отображение, в которое будет происходить проваливание при клике на запись отображения (для отличных от Тип отображения==DETAILS отображений)
  • Поле со значение ключа, Поле выбора экрана одной записи - идентификатор записи в таблицы и указание экрана одной записи
  • Поле иерархии - для Типа отображения TREE_LIST указание родительского элемента
  • Группировка по полям, Сортировка по полям - только для отображений LIST, указывает поля, по которым будет происходить группировка и сортировка
  • Добавление, Обновление, Удаление - операции inline редактирования для отображений типа LIST
  • Добавление через модальное окно - операция добавления по средствам формы одной записи
  • Для отображений, отличных от SIMPLE_LIST, LIST, необходимо будет заполнить дополнительные поля, подробнее в Создание и редактирование отчетов

 4. После заполнения необходимых полей нажмите кнопку Сохранить
