В открывшейся форме заполните поля (обязательные поля подсвечиваются красным цветом)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  

Имя (название формата поля для пользователей системы).

Тип поля (тип данных, в котором будут храниться данные в полях с создаваемым форматом поля):

  • INTEGER (целое число).
  • DOUBLE (вещественное число).
  • BOOLEAN (логическое да/нет).
  • STRING (строка до 255 символов).
  • TEXT (строка более 255 символов).
  • DATETIME (дата и время).
  • DATE (дата).
  • TIME (время).
  • VIRTUAL (виртуальное поле – не хранит в себе данных, является ссылкой на данные, подробнее см. раздел Добавление и редактирование виртуальных полей).

Тип шаблона (описание механизма получения данных): 

  • attachment (файл-одиночное вложение).
  • attachmentList (массовое вложение).
  • lookupEntity (ссылка на данные из какого-либо объекта системы).
  • lookupList (из указанных в виде списка значений, в поле Шаблон необходимо перечисление возможных вариантов с разделителем «;», например: «Доработка;Баг;Поддержка;Не указано» или «1;2;3;4;5»).
  • lookupQuery (в будущем будет отображать в качестве значений результат произвольного sql-запроса).
  • range (диапазон значений). В поле «Шаблон» можно указать допустимые границы в формате массива JavaScript. Если одна из границ не нужна, укажите для нее null,
    Например: значение «> 0» будет выглядеть так [0, null].
  • regexp(Регулярное выражение). В поле «Шаблон» можно указать регулярное выражение в формате JavaScript (подробнее о регулярных выражениях см. здесь). 
    Пример регулярного выражения:
«^\w+\s\w+$»


       где «^» обозначает начало входных данных
      «\w»- любой словесный (латинский алфавит) символ, включая буквы, цифры и знак подчеркивания, что эквивалентно [A-Za-z0-9_]
       «+» - повторение символа 1 или более раз,
       «\s» - любой пробельный символ, включая пробел, табуляцию, переводы строки и другие юникодные пробельные символы,
       «$» -  конец входных данных

  • report (ссылка на объект отчет).
  • vjAddOne (виртуальное поле – таблица данных из какого-либо объекта системы с возможностью добавления записей).
  • vjReadOnly (виртуальное поле таблица данных из какого-либо объекта системы).
  • vjLink (виртуальное поле – таблица данных из какого-либо объекта системы с возможностью добавления записей и возможностью связывания с уже существующими записями).

Стиль поля (определяет внешний вид поля (виджет) на форме просмотра одной записи): 

  • attachment (для типа шаблона attachment).
  • attachmentList (для типа шаблона attachmentList).
  • calendar (для полей, ссылающихся на объект Календарь системы, в таких случаях рекомендуется не создавать новый, а использовать уже существующий в системе формат поля «Календарь» (внутренний номер 523) и указывать внутренний номер объекта Календарь в поле «Значение по умолчанию» описания атрибута объекта).
  • number (для типа поля INTEGER).
  • text (для типа поля STRING).
  • textarea (для типа поля TEXT).
  • date (для типа поля date).
  • datetime (для типа поля datetime).
  • time (для типа поля time).
  • checkbox (для типа поля BOOLEAN).
  • float (для типа поля DOUBLE).
  • combobox (комбинированный список, для типов шаблона lookupEntity, lookupList).
  • autocomplete (для типов шаблона lookupEntity, lookupList с возможностью проваливания в связанную запись).
  • addable_combobox (аналогично стилю combobox, но с возможностью добавления).
  • addable_autocomplete (аналогично стилю autocomplete, но с возможностью добавления новых значений в справочник).
  • simple_subgrid (для типа поля VIRTUAL, отображает список записей в системе без функций группировки и изменения состава полей).
  • subgrid (для типа поля VIRTUAL, отображает список записей в системе).
  • email (поле будет проверяться на соответствие шаблону в формате адреса электронной почты).
  • url (для полей, в которых будет храниться какая-либо ссылка).
  • report более не используется.
  • complex (для поля с изменяющимся форматом поле, текущее значение формата определяетcя значением поля со следующей маской: <Название поля>_FORMAT, в таких случаях рекомендуется не создавать новый, а использовать уже существующий в системе формат поля «Плавающий формат» (внутреннее имя «complex_template»)).
  • password (для полей, в которых будет храниться какой-либо пароль).
  • html (для полей, в которых нужно отображать некий текст, оформленный с помощью средств html-меток; в таких случаях рекомендуется использовать уже существующий формат поля: «HTML метка» (внутреннее имя «html_template»)).
  • code (для типа поля TEXT, в случае, если поле будет содержать какой-либо код).
  • richtext (текст с расширенными возможностями форматирования).
  • barChart (для типа шаблона report).
  • pieChart (для типа шаблона report).

Шаблон (шаблон выражений для типов шаблона LookupList, range, regexp).

Таблица (для типа шаблона LookupEntity – указывает на таблицу из записей которой будут получаться данные).

Поле с хранимым значением (для типа шаблона LookupEntity – указывает поле из Таблицы, значение которого будет записываться в поле с создаваемым форматом поля).

Поле с отображаемым значением (для типа шаблона LookupEntity – указывает поле из Таблицы, значение которого будет отображаться для выбора в поле с создаваемым форматом поля).

Связанный экран - экран, который будет использоваться при отображении данного поля

Внутреннее имя - системное имя, присваивается объекту после его создание

Относится к рабочему пространству - рабочее пространство, в котором данный формат поля будет доступен, по умолчанию проставляется текущее рабочее пространство пользователя

"isActive=1 and " + FieldFormatUtils.getLookupEntityFieldQuery(record.getAsString("lookupEntity"))
