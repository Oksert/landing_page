Для того, чтобы сгенерировать новый модуль с уже преднастроенной логикой нужно:

1) Загрузить в систему все UNL из этого файла (можно сделать это удалив все файлы из папки ...\Server\data, вставив 500+ файлов в эту же папку и вызвав configure.bat залить в пустую БД, второй вариант загрузки в ps)

2) Сгенерить новую логику

1. Загружаешь выгрузки

2. Копируешь dbdict krexample + dbdict для истории krhistorykrexample

3. Находишь либу KRLogicGenerator

4. В либе в вызове функции CopyObjectLogic(targetObjectName) пишешь название нового dbdict'а

5. Нажимаем "Выполнить"

6. После нужно в процессе <dbdictName>.open.for.add на вкладке "Исходный JS" нужно изменить аргумент вызова new SCFile("") на <dbdictName>

3) Удалить объекты krexample используя скрипт выгрузки KR UNL EXAMPLE LOGIC


P.S. Второй вариант загрузки unl: берем здесь дамп бд, восстанавливаем, в таблице infooldm1 удаляем строку, содержащую тип *scdb*, создаем нового пользователя БД и ставим ему следующие мэппинги (картинка ниже): public, db_datareader, db_datawriter, db_ddladmin. Создаем odbc драйвер и далее после настройки sm.ini через configure.bat как обычно подключаемся через клиент.
