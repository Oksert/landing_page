Для загрузки данных в систему из Excel выполните следующие действия:

  1. Подготовьте данные в шаблоне для загрузки согласно следующих правил:
    • Файл должен быть в формате .xlsx с одной вкладкой.
    • Форматирование по типам данных колонок во вносимом файле должно соответствовать типам данных объекта системы.
    • В качестве заголовков у колонок должно быть прописано имя поля объекта системы.
    • Все столбцы, указанные в заголовках, должны быть созданы как поле объекта, в который загружаются данные. Если в системе столбец отсутствует, то его необходимо создать, для этого:
      • Пройдите по ссылке «http://[serverhostname]/#/entitytype», откройте информацию об объекте, в который будут загружены данные, откройте секцию «Поля».
      • Нажмите кнопку «Добавить запись», в открывшемся окне заполните все необходимые поля (выделены красным цветом) и нажмите «Создать поле», после чего нажмите «Готово». Для каждого поля, которое необходимо создать повторите данные действия.

Пример шаблона файла доступен по ссылке: «https://goo.gl/YARv4F».

  1. Опишите правила обработки каждой строки из исходного файла (определяется необходимость добавления новой записи/обновления одной из существующей записи/пропуск записи). Для этого выполните следующие действия:
    • Открыть в браузере страницу системы «http://[serverhostname]/#/entityreconciliation» - «Правила Реконсиляции (ENTITYRECONCILIATION)».
    • Добавить запись для каждого поля соответствующего типа, заполнить поля следующим образом:
      • Порядок сортировки (очередной порядковый номер правила для объекта системы, данные по которому будут загружены из Excel-файла (задают порядок проверки правил)).
      • Объект реконсиляции (объект системы, данные по которому будут загружены из Excel-файла).
      • Условие срабатывания (код на языке Java, описывающий условия срабатывания создаваемого правила (для безусловного выполнения – «true»)).
      • Тип реконсиляции («QUERY», если необходимость создания/обновления записи определяется набором полей и «FIELD», если одним полем).
      • Стратегия слияния (Определяет способ слияния текущей записи с загружаемой. Значение «0» - если в случае удовлетворения условия реконсиляции требуется сохранить текущие данные без изменений, «1» - если в случае удовлетворения условия реконсиляции требуется полностью перезаписать текущие данные значениями вносимых данных, «2» - если в случае удовлетворения условия реконсиляции требуется обновить текущие данные только в пустых полях).
      • Поле реконсиляции (заполняется только когда указан тип реконсиляции «FIELD», указывает используемое поле объекта реконсиляции).
      • Запрос реконсиляции (код на языке Java, описывающий правило определения уникальности записей, вносимых в систему; заполняется, когда указан тип реконсиляции «QUERY»).

Пример запроса реконсиляции:

"entityTypeId=" + record.get("entityTypeId") + " and fieldName='" + record.get("fieldName") + "'"

В данном примере:

entityTypeId- название поля объекта в системе

record.get("entityTypeId") - название поля во вносимом Excel-файле (после обработки поля правило реконсиляции поля)

    3. Если в исходном файле для полей типа LookupEntity, LookupList (поля которые имеют различные хранимые и отображаемые значения) указано отображаемое значение поля, то для каждого такого поля создайте правила реконсиляции (это необходимо для корректной записи внутреннего кода). Для этого нужно выполнить следующие действия:

  • Открыть страницу системы «http://[serverhostname]/#/FIELDRECONCLIACTION» - Реконсиляция значения поля (FIELDRECONCLIACTION).
  • Нажать на кнопку "Добавить запись" и заполнить открывшуюся форму следующим образом:
    • Код сущности (объект системы, данные по которому будут загружены из Excel-файла).
    • Порядок сортировки (очередной порядковый номер правила для объекта системы, данные по которому будут загружены из Excel-файла (задают порядок выполнения правил)).
    • Имя поля (имя поля в объекте системы, данные по которому будут загружены из Excel-файла).
    • Условие срабатывания (код на языке Java, описывающий условия срабатывания создаваемого правила (для безусловного выполнения - "true")).
    • Скрипт реконсиляции (resultValue) (код на языке Java, описывающий правило сопоставления внутреннего кода и значений полей, вносимых в систему).

Пример скрипта реконсиляции:

Object resultValue = null;

EntityDTO obj = QueryUtils.getRecord("R_ORGUNIT", "DISPLAYNAME='" + fieldValue + "' and  ORGUNITTYPE='department'");

if (obj != null){

        resultValue = obj.getAsInteger("R_ORGUNITID");

}

В данном коде используются следующие переменные:

fieldValue - значение поля, которое пришло из исходного файла;

resultValue - конечное значение поля, которое будет сохранено в систему.

Для того, чтобы обратиться к другому полю загружаемой строки можно использовать переменную inputLine, ниже пример кода:

Object resultValue = null;
EntityDTO obj = QueryUtils.getRecord("SM_CIMODELTYPE", "DISPLAYNAME='" + fieldValue + "'");

     resultValue = obj.getAsInteger("SM_CIMODELTYPEID");

   4. Загрузите данные непосредственно в систему, для этого:

  • Пройдите по ссылке «http://[serverhostname]/#/loadentitytype», нажмите кнопку «Добавить запись», заполните следующие поля:
    • Объект (объект системы, в которой будут загружаться данные из Excel-файла).
    • Файл для загрузки (укажите Excel-файл для загрузки).
    • Статус загрузки («pending»).
  • Нажмите "Создать", затем "Загрузить данные".

В случае корректности данных произойдет загрузка данных, в случае наличия ошибок - в поле «Лог загрузки» появится сообщение с указанием конкретной ошибки загрузки.

   5. Проанализируйте протокол загрузки данных.

В результате успешной загрузки в поле «Лог загрузки» появится сообщение, в котором будет видно количество добавленных, обновленных и пропущенных записей из загружаемого файла, время начала и окончания загрузки:

INFO: Загрузка начата в 2015/01/23 07:22:22

INFO: Загрузка завершена в 2015/01/23 07:22:47:

Добавлено записей: 60

Обновлено записей: 2

Пропущено записей: 1

В случае наличия ошибок в логе будет строка следующего вида:

ERROR: Line 14391. java.lang.Exception: Ошибка при реконсиляции значения поля. Поле 'RAILWAY', значение: 'Западно-Казахстанская'

В этом случае необходимо проверить значение соответствующей строки (в приведенном примере «14391») в загружаемом файле на предмет корректного выполнения условий, описанных в пункте 1 данной инструкции. После исправления данной ошибки в исходном файле необходимо повторить операцию загрузки данных в систему.
