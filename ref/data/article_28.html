В систему можно загрузить данные из Excel-файла. За один раз можно загрузить данные только одной Таблицы. 

Для этого нужно последовательно выполнить следующие шаги:

  • Подготовить шаблон для загрузки данных в формате Excel-файла
  • Создать правило реконсиляции объекта - правила обработки строки файла (создание нового объекта или обновление уже существующего объекта )
  • Если требуется, то создать правила реконсиляции полей - правила по которым на основании видимых значений поля будет вычислено хранимое значение. Например, это нужно когда ФИО пользователя "Иванов Иван Иванович" нужно при загрузке заменить на ID пользователя. 
  • Загрузить подготовленный файл в Систему
  • При наличии ошибок в логе загрузки внести изменения в исходные данные и повторить загрузку

В данной статье подробно описаны действия для выполнения каждого из шагов..

Подготовка Excel-файла

Подготовьте данные в шаблоне для загрузки согласно следующим правилам:

  • Файл должен быть в формате .xlsx с одной вкладкой.
  • Форматирование по типам данных колонок в файле для загрузки должно соответствовать типам данных объекта системы.
    Для того, чтобы загрузить данные типа Строка, содержащие числовые данные нужно средствами Excel привести весь столбец к типу строка (в левом верхнем левом углу ячейки должен быть зеленый треугольник). Например, для этого можно использовать функцию ТЕКСТ Excel или функций Разделить текст по столбцам, не указывая разделитель и указав тип исходных данных Текст.
    Для того, чтобы загрузить данные типа Дата нужно установить формат поля строка, а данные записать в формате DD/MM/YY.
  • В качестве заголовков у колонок должно быть прописано Системное имя поля объекта системы.
  • Все столбцы, указанные в заголовках Excel-файла, должны быть созданы как поля Таблицы, в который загружаются данные. Если в системе столбец отсутствует, то его необходимо создать (о том как создать поле смотрите в статье Добавление и редактирование полей)

Создание правил реконсиляции Таблиц

Правила реконсиляции Таблиц (объектов) определяют правило обработки каждой строки загружаемего файла.

На основании правила система определяет, что сделать со строкой:

  • Добавить как новый объект
  • Обновить существующий объект
  • Пропустить запись.
  • Для этого необходимо выполнить следующие действия:

Для создания правила реконсиляции объекта перейдите к пункту меню Настройка -> Загрузка данных -> Правила реконсиляции для таблиц

В открывшемся списке нажмите на кнопку +Добавить запись и в открывшейся форме заполните поля:

  • Объект реконсиляции - объект системы, данные по которому будут загружены из Excel-файла.
  • Условие срабатывания - код на языке Java, описывающий условия срабатывания создаваемого правила (для безусловного выполнения – «true»). Для обращения к полям загружаемой записи используется выражение, следующего вида:  record.get("<название поля>").
    Пример условия срабатывания:

    • record.get("fieldName")!=null

    Где record.get("entityTypeId") - название поля в Excel-файле (после обработки поля правило реконсиляции поля)

  • Тип реконсиляции - необходимо указать значение «FIELD», если у объекта есть однозначное поле реконсиляции, по которому можно проверять создавать новый объект или обновлять существующий.  Укажите значение  «QUERY»,  если необходимость обновления или создания определяется совокупностью полей или еще более гибким условием.

  • Стратегия слияния - определяет способ обновления текущей объекта с загружаемыми данными. Значение «0» - если в случае удовлетворения условия реконсиляции требуется сохранить текущие данные без изменений, «1» - если в случае удовлетворения условия реконсиляции требуется полностью перезаписать текущие данные значениями вносимых данных, «2» - если в случае удовлетворения условия реконсиляции требуется обновить текущие данные только в пустых полях.
  • Поле реконсиляции - заполняется только когда указан тип реконсиляции «FIELD», указывает используемое поле объекта реконсиляции.
  • Запрос реконсиляции - запрос на языке SQL, проверяющий существует ли загружаемая строка в системе. Если в результате применения запроса возвращаются данные, то первая из найденных строк обновится, если нет, то будет создана новая запись.
    Пример запроса реконсиляции:
    "entityTypeId=" + record.get("entityTypeId") + " and fieldName='" + record.get("fieldName") + "'"

    Где:

    entityTypeId- название поля объекта в системе

    record.get("entityTypeId") - название поля в загружаемом Excel-файле (после обработки поля правило реконсиляции поля)

  • Порядок сортировки  -  номер правила для объекта системы, данные по которому будут загружены из Excel-файла (задают порядок проверки правил). Необходимо, если для одного объекта задано более одного правила
    • Где record.get("entityTypeId") - название поля в загружаемом Excel-файле (после обработки поля правило реконсиляции поля)
    Пример созданного правила реконсиляции:                                                                                                                                                                      

    Создание правил реконсиляции полей объекта

    Если в исходном файле для полей типа LookupEntity, LookupList (поля которые имеют различные хранимые и отображаемые значения) указано отображаемое значение поля, то для каждого такого поля создайте правила реконсиляции (это необходимо для корректной записи id объекта).

    Для этого перейдите к пункту меню Настройка -> Загрузка данных -> Правила реконсиляции для полей

    В открывшемся списке нажмите на кнопку  +Добавить запись  и в открывшейся форме заполните поля:

    • Код сущности (объект системы, данные по которому будут загружены из Excel-файла).
    • Порядок сортировки - определяет порядок выполнения правил.
    • Имя поля (имя поля в объекте системы, данные по которому будут загружены из Excel-файла).
    • Условие срабатывания (код на языке Java, описывающий условия срабатывания создаваемого правила (для безусловного выполнения - "true")).
    • Скрипт реконсиляции - код на языке Java, описывающий правило сопоставления внутреннего кода и значений полей, вносимых в систему. В результате выполнения скрипта должно быть рассчитано значение переменной resultValue.

    Пример скрипта реконсиляции, который на основании наименования подразделения, указанного в строке загружаемого файла рассчитывает его id:

    Object resultValue = null;
EntityDTO obj = QueryUtils.getRecord("R_ORGUNIT", "DISPLAYNAME='" + fieldValue + "' and  ORGUNITTYPE='department'"); 
if (obj != null){
   resultValue = obj.getAsInteger("R_ORGUNITID");
}


    В данном коде используются следующие переменные:

    fieldValue - значение поля, которое пришло из исходного файла;

    resultValue - конечное значение поля, которое будет сохранено в систему. 

    Загрузка данных

    После того как исходный файл подготовлен и созданы необходимы правила реконсиляции объекта и полей объекта его можно загрузить в Систему.

    Для этого  перейдите к пункту меню Настройка -> Загрузка данных -> Загрузка из Excel

    В открывшемся списке нажмите на кнопку  +Добавить запись  и в открывшейся форме заполните поля:

    • Объект - объект системы, в которой будут загружаться данные из Excel-файла.
    • Файл для загрузки - укажите Excel-файл для загрузки.

    После заполнения полей нажмите на кнопку Создать, затем Загрузить данные.

    В случае корректности данных поле Статус загрузки примет значение "SUCCESS", все данные окажутся в системе, а в поле Лог загрузки появится информация о количестве загруженных и обновленных данных.

    В случае наличия ошибок во всем файле или хотя в одной строке исходного файла поле Статус загрузки примет значение "FAILURE" и в поле Лог загрузки появится сообщение с указанием конкретной ошибки загрузки, при этом строки с корректными данными будут загружены в систему.

    После завершения каждой загрузки в поле Лог загрузки появится сообщение, в котором будет видно количество добавленных, обновленных и пропущенных записей из загружаемого файла, время начала и окончания загрузки:

    INFO: Загрузка начата в 2016/01/23 07:20:46

    INFO: Загрузка завершена в 2016/01/23 07:22:47

    Добавлено записей: 60

    Обновлено записей: 2

    Пропущено записей: 1

    В случае наличия ошибок в поле Лог загрузки будет строка следующего вида:

    ERROR: Line 14391. java.lang.Exception: Ошибка при реконсиляции значения поля. Поле 'RAILWAY', значение: 'Западно-Казахстанская'

    В этом случае необходимо проверить значение соответствующей строки (в приведенном примере «14391») в загружаемом файле на предмет корректного выполнения правила реконсиляции поля RAILWAY, а также убедиться, что все данные, которые указаны в столбце RAILWAY исходного файла есть в соответствующем справочнике системы.

    После исправления данной ошибки в исходном файле необходимо повторить операцию загрузки данных в Систему.
