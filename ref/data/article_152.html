Для установки системы на  ОС Windows и СУБД PostrgreSQL необходимо выполнить следующие шаги:

  • Подготовить сервер (физическая или виртуальная машина), удовлетворяющую требования к программному и аппаратному обеспечению
  • Установить и настроить СУБД PostgreSQL, восстановить дамп базы данных системы
  • Установить Java 8
  • Установить и настроить приложение IdeaPlatform

 Ниже подробно описано выполнение каждого шага установки системы.

Минимальные требования к программному и аппаратному обеспечению

Минимальные требования к аппаратному обеспечению:

2 CPU, 4 Gb RAM, 5 Gb HDD

Требования к программному обеспечению:

ОС: Windows Server 2008R2 или Widows Server 2012R2 с последними обновлениями

Установка PostgreSQL 9.6

Скопировать файл postgresql-9.6.3-1-windows-x64.exe в папку C:\temp  и запустить установщик postgresql-9.6.3-1-windows-x64.exe следуя инструкциям:

Добавить путь «C:\Program Files\PostgreSQL\9.6\bin» в переменную среды Path

Путь Control Panel\System and Security\System -> Advanced System Settings -> Environment Variables

Восстановление дампа

Распаковать архив ipdb.zip в файл C:/temp/ipdb.sql. 

Запустить командную строку с правами администратора и  выполнить команду: 

psql -U postgres

Запустить команду восстановления базы из дампа:

\i C:/temp/ipdb.sql

Установка Java 8

Скопировать файлы jre-8u131-windows-x64.exe и jdk-8u131-windows-x64.exe в папку C:/temp/

Запустить установщики jre-8u131-windows-x64.exe и jdk-8u131-windows-x64.exe, проследовав  всем инструкциям (без изменения пути установки)

Добавить переменную JAVA_HOME в переменные среды: 

JAVA_HOME=C:\Program Files\Java\jdk1.8.0_144

Установка приложения Idea Platform

Распаковать архив с дистрибутивом приложения IdeaPlatform_SAAS_Distr.zip в папку C:\WebServers\wildfly-8.2.1.Final

Запустить командную строку с правами администратора и выполнить следующие команды:

cd C:\WebServers\wildfly-8.2.1.Final\bin\service
service.bat install 

Настройка приложения Idea Platform

Настройка приложения выполняется путем редактирования параметров запуска системы в файле C:\WebServers\wildfly-8.2.1.Final\standalone\configuration\system.properties. 

В файле system.properties необходимо задать параметры javaHomeJDK и javaHomeJRE:

javaHomeJDK=C:/Program Files/Java/jdk1.8.0_144
javaHomeJRE=C:/Program Files/Java/jre1.8.0_144

Полное описание параметров смотрите в статье Описание параметров файла конфигурации system.properties

Ниже приведен пример файла system.properties для ОС Windows:

dbConnectionUrl=jdbc:postgresql://127.0.0.1:5432/ip_dev
dbSystemSchema=system
dbUserName=system_user
dbPassword=P@$$w0rd
javaHomeJDK=C:/Program Files/Java/jdk1.8.0_144
javaHomeJRE=C:/Program Files/Java/jre1.8.0_144
deploymentFolder=deployments/IdeaPlatform_SAAS.war
reindexTextOnStartup=false
compileClassLibraryOnStartUp=false
traceHttpRequestTime=true
baseURL=http://localhost:8080

Настройка подключения к базе данных

Если сервер приложений и СУБД находятся на одном физическом или виртуальном сервере, то дальнейшая настройка не требуется.

Если сервер приложений и СУБД находятся на разных серверах, то необходимо настроить подключение сервера приложений к СУБД по JDBC.

Необходимо задать параметр dbConnectionURL с учетом IP адреса сервера СУБД, а

dbConnectionUrl=jdbc:postgresql://127.0.0.1:5432/ip_dev

Открыть возможность подключения к серверу СУБД извне:

  • На сервере СУБД перейти в папку C:\Program Files\PostgreSQL\9.6\data

  • Открыть на редактирование файл postgresql.conf и задать значение параметра

    listen_addresses = '*', сохранить файл.

  • Открыть на редактирование файл pg_hba.conf и разрешить подключение для любых пользователей с IP адреса сервера приложений. Для этого нужно найти строку с префиксом IPv4 local connections и задать в колонке ADDRESS IP адрес сервера приложений и METHOD md5: 

    # TYPE  DATABASE        USER            ADDRESS                 METHOD
# IPv4 local connections:
host    all             all                        md5

    Далее необходимо сохранить файл.

  • Перезапустить службу postgresql-x64-9.6

Вход в систему

Для входа в систему необходимо открыть браузер и перейти по ссылке указанной в параметре baseURL ( подробнее о параметр baseURL смотрите описание файла system.properties). По умолчанию ссылка на систему имеет следующий вид:

http://:8080

где - это доменное имя или IP сервера приложений. 

На открывшейся странице входа введите следующие параметры:

  • Логин: adm@runidea.online
  • Пароль: admin
