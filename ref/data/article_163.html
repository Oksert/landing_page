Отчеты типа Гистограмма (STACKED_BARCHART)  представляет собой графический отчет, позволяющие анализировать вхождение категорий данных в группы.

Например, количество сотрудников определенных должностей в командах компании (смотрите пример ниже).

На гистограмме можно отображать различные серии данных (в примере должности), которые будут различаться по цвету. Также есть возможность объединения нескольких серий, отображаемых в виде столбчатых диаграмм, в группу, которая отображаются в виде одного столбца (в примере это одна команда), разбитого на несколько полос.

Создание Таблицы для отчета типа «Гистограмма»

Перейдите к пункту меню Настройка -> Описание таблиц

На открывшейся странице нажмите кнопку +Добавить запись.

В открывшейся форме заполните обязательные поля:                                                                                                                                                                                        

Название - название таблицы

Системное имя - имя объекта на латинице, которое будет использоваться в бизнес-логике                                                                                                                             

Тип источника данных - укажите значение Запрос

Sql Query - запрос к базе данных на языке SQL, который обязательно содержит следующие поля: 

  • XVALUE - значение по оси X графика. Обязательно должно иметь тип строка. Значения поля будут отображаться на оси X
  • YVALUE - значение по оси Y графика.  Обязательно должно иметь числовой тип данных, как правило это Целое или Вещественное число
  • SERIES - наименование серии. Точки с одной серии будут отображаться одним цветом
  • GROUP_NUMBER - наименование группы. Точки, принадлежащие графику типа "Столбчатая диаграмма", входящие в одну группу будут отображены как столбики, стоящие друг на друге. Как правило значение поле совпадает со значение поля XVALUE
  • COLOR_FIELD - цвет серии. Должно содержать значение цвета в формате шестнадцатеричного кода с использованием маски #RRGGBB или один из стандартных HTML-цветов (red, blue, green)

Пример запроса:

select 'команда 1' as xvalue, 200 as yvalue, 'Аналитик' as series, 'команда 1' as group_number, 'green' color_field
union all
select 'команда 1' as xvalue, 300 as yvalue, 'Разработчик' as series, 'команда 1' as group_number, 'red' color_field

union all
select 'команда 3' as xvalue, 120 as yvalue, 'Аналитик' as series, 'команда 3' as group_number, 'green' color_field
union all
select 'команда 3' as xvalue, 110 as yvalue, 'Разработчик' as series, 'команда 3' as group_number, 'red' color_field 
union all
select 'команда 3' as xvalue, 40 as yvalue, 'Бизнес' as series, 'команда 3' as group_number, 'blue' color_field

union all

Создание отчета типа Гистограмма (STACKED_BARCHART)

Для создания отчета типа "Гистограмма" (STACKED_BARCHART) перейдите к пункту меню Настройка->Настройка отображения.

Нажмите на кнопку + Добавить запись и в открывшемся окне заполните поля  следующим образом:

Название - название отчета

Таблица - таблица данные которой нужно отобразить. Укажите таблицу, которая была создана в первом пункте данной статьи

Системное имя - наименование на латинице, которое будет использовано в качестве ссылки на отчет. Таким образом ссылка на создаваемый отчет будет иметь вид: http://:/#/<Системное имя>
где   - имя или ip сервера, на котором установлен web application server системы
          - порт, на котором установлен web application server системы
Например, для отчета с системным именем "my_tasks" ссылка будет иметь вид:  http://runidea.online/#/my_tasks

Тип отображения - для отображения типа "Графика" необходимо указать тип STACKED_BARCHART

Запрос для выборки данных - условие на языке SQL к таблице указанной в поле Таблица, которому должны удовлетворять все записи, попадающие в отчет.  Важно отметить, что к полям объекта нужно обращаться через их системное имя, также как в API системы. Также в запросе можно использовать переменные среды и значения полей поиска текущего отчета, для этого перед полем поиска нужно поставить символ @.

Пример запроса:

assignedto = @currentUserid and workflowstepname = 'inwork' and type = '@p_type'


В данном примере используются следующие переменные:

  • @currentUserid - переменная среды, которая хранит id текущего пользователя системы
  •  @p_type - это ссылка на поле Поиска данного отчета. О том как создать поле поиска и задать ему значение по умолчанию читайте в статье Настройка полей поиска.

Шаблон заголовка - укажите текст, который будет отображаться в заголовке отчета. В заголовке отчета могут быть использованы ссылки на значения полей поиска. Для этого системное имя поля поиска нужно взять в двойные квадратные скобки [[имя поля поиска]].
Например, если отчет отображает задачи определенного типа, значения которого указано в поле поиска p_type, то заголовок будет выглядеть следующим образом:

Задачи типа [[p_type]]

Имя серии - укажите поле Таблицы, в котором хранятся серии точек (в примере выше это поле series)

Значение по X - укажите поле Таблицы, в котором хранятся координаты по оси  X (в примере выше это поле xvalues)

Значение по Y - укажите поле Таблицы, в котором хранятся координаты по оси  Y (в примере выше это поле yvalues)

Номер группы - укажите поле Таблицы, в котором хранятся группы значений (в примере выше это поле group_number)

Цвет серии - укажите поле Таблицы, в котором хранится цвет серии (в примере выше это поле color_field)

После заполнения всех полей нажмите на кнопку Cоздать
