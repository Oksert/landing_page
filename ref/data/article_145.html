Описание функционала

Данное приложение позволяет пользователю с ролью Менеджер создавать заказы, указывая клиента, механика, на которого оформлен заказ и список деталей. После указания данных автоматически рассчитывается стоимость заказа.

Менеджер имеет возможность изменять Механика и Клиента для каждого конкретного заказа, что будет отображено в истории заказа. У Менеджера будет создан графический отчет, показывающий распределение заказов по механикам, при клике на область которого будет отображена более детальная информация по каждому механику.

Для каждого механика будет настроено свое рабочее место и отчеты:

  • Отчет, показывающий все заказы, статус ЖЦ которых отличен от Выполнен и которые оформлены на данного механика. Данные могут быть отфильтрованы по каждому из полей отчета
  • Отчет заказов за период, содержащий все заказы в указанном временном интервале на данного Механика

У каждого пользователя будет возможность осуществлять глобальный поиск по текстовым полям и массовая кнопка, позволяющая перевести все отмеченные заказы в статус Новый Жизненного Цикла Заказа.

Настройка приложения

Настройка приложения состоит из следующих шагов :

1. Создание объектов и веб-интерфейса для редактирования объектов

2. Настройка жизненных циклов объектов

3. Настройка отчетов и рабочих мест пользователей

Создание объектов системы

На данном этапе мы создадим объекты и настроим интерфейс для просмотра и редактирования:

  1. Создаем объект Клиент с полями: Телефон, Почта, Наименование.
  2. Добавляем возможность редактирования данных по клиентам прямо из таблицы  (инлайн-редактирование)
  3. Настраиваем форму списка и форму одной записи объекта клиент
  4. Создаем объект Заказ деталей
  5. Добавляем поля Клиент (в формате поля указав таблицу Клиент), Механик (создадим новый формат поля типа, autocomplete, который будет выбирать только те записи из таблицы Пользователь, у которых roleid=15), Число затраченных часов, Стоимость заказа, Описание
  6. Включаем ведение истории изменения атрибутов, возможность задать для объекта Жизненный цикл и поиск по текстовым полям объекта (установим галки Поиск по текстовым полям, Аудит и Workflow)
  7. Создаем объект Запчасти, добавим поля Описание, Цена. Включим ведение истории и поиск по текстовым полям (установим галки Аудит и Поиск по текстовым полям)
  8. Настраиваем форму списка и форму одной записи объекта Запчасти (для этого кликаем на знак гаечного ключа переходим в редактор формы одной записи)
  9. Создаем объект Детали заказа, добавляем поля Заказ (запись таблицы Заказ деталей), Деталь (запись таблицы Запчасти), Цена
  10. Настраиваем форму списка и форму одной записи объекта Детали заказа
  11. Настроим отображение деталей в заказе. Для этого создадим виртуальное поле, содержащее все записи таблицы Детали заказа, у которых значение поле Заказ совпадает с id текущей записи
  12. Настроим права доступа к объекту Детали Заказа. Переходим в секцию Карты доступа, создаем новую запись, разрешая пользователям, которые стоят в поле Механик обновлять запись
  13. Настроим форму одной записи объекта Заказ деталей.
  14. Переходим в настройки отображения виртуального поля Детали, добавляем возможность редактирования прямо из списка  (инлайн-редактирования)
  15. Создаем правил, модифицирующее название Деталей заказа таким образом, чтобы в нем отражался Заказ. Прикрепляем к таблице Запчасти
  16. Создаем правило, формирующее Стоимость заказа по сумме запчастей, входящих в заказ

Добавление жизненного цикла к объекту

На данном шаге мы настроим жизненный цикл, описывающие бизнес-процесс обработки объекта Заказ.

Первым делом, мы добавляем шаги жизненного цикла (состояния, в которых может находится объект).

Жизненный цикл Заказа состоит из 3-х шагов:

  • Новый
  • В работе
  • Исполнен

Далее создаем возможные переходы из шага в шаг и даем права доступа на эти переходы определенным пользователем :

  • Переход из шага "Новый" в шаг "В работе" назовем В работу. Даем права на данный переход любом пользователю , входящему в группу доступа с id = 1
  • Переход из шага "В работе" в шаг "Исполнен" назовем Исполнен. Даем права на данный переход любом пользователю , входящему в группу доступа с id = 1

Теперь привяжем созданный жизненней цикл к объекту Заказ:

  • Перейдем в Настройки объекта Заказ деталей
  • В поле Work Flow укажем id созданного нами ЖЦ
  • В поле Статус укажем значение по умолчанию равное  id первого шага созданного нами ЖЦ
  • В поле Название шага укажем значение по умолчанию равное названию первого шага созданного нами ЖЦ  

Создание отчетов и настройка рабочих мест пользователей

На данном шаге мы создадим следующие отчеты:

  • Заказ деталей, отфильтрованный по шагу жизненного цикла.
  • Распределение заказов по механикам. Добавляем дочерний отчет, параметром которого будет являться Механик. Дочерний отчет будет представлять собой список, состоящий из заказов данного механика

Для добавления дочернего отчета:

  • Перейдем в таблицу Связь настроек отображения, укажем в качестве родительского экрана отчет Распределение заказов по механикам, в качестве дочернего отчета Заказ деталей по механику
  • Для передачи полей перейдем в секцию Меппинг полей и добавим запись, указав поле дочернего отображения, Способ передачи - PARENT_FIELD, указывающий то, что поле должно браться из родительского отображения, ID связи, созданной ранее и Родительское отображение

Далее создадим рабочее место для пользователя с ролью Механик, для этого:

  • Перейдем в таблицу Меню, создадим новую запись Панель управления механика
  • В секции Дочерние отображения добавим две новые записи Создать, Найти, Отчеты в поле Тип указывает group, так как они буду содержать другие пункты меню
  • Для каждой из созданной Найти и Создать записей создаем дочерние отображения с типом item, соответствующие Заказу и Детали
  • Для пункты Отчеты создаем дочерние пункты:
    • Заказы на исполнении, в поле Ссылка указываем адрес отчета в системе, Порядок сортировки 1
    • Распределения заказов по механикам, аналогично предыдущему пункту, Порядок сортировки 2
  • Идем в таблицу Профиль пользователя, выбираем запись Механик
  • В поле Меню выбираем созданное нами меню
  • Настроим видимость полей объекта для пользователя механик
