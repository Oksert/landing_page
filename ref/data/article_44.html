Для редактирования схемы жизненного цикла:

  • Перейдите в меню Настройка - Жизненные циклы
  • Выберите нужный вам, кликните 2 раза по записи
  • В открывшемся окне нажмите кнопку Редактировать workflow, откроется редактор схемы ЖЦ     

Создание шага:

  • Перетащите иконку состояния Active State из правой панели шаблонов состояний в область редактирования workflow.                                                                     

                                                                                                                                                                                                                                                                                                                                                                                                                     

                                                                                                                                             Добавление нового шага жизненного цикла

  • В открывшемся окне заполните поля (обязательные подсвечиваются красным):
    • Имя (отображаемое имя состояния);
    • Код шага (системное имя состояния).
    • Sort order - определяет порядок кнопки перехода на форме

                                                                                                                                                                                                                                                                                                                                                                                                                          

                                                                                                                                               Определение нового шага жизненного цикла

  • Нажмите кнопку Создать, затем Готово.
  • Аналогичным образом создайте все остальные шаги из создаваемого жизненного цикла.

Создание перехода workflow:

  • Нажмите левой кнопкой мыши в зону красного круга в правой области начального состояния и не отпуская проведите курсор в зону зеленого круга в левой области конечного состояния, в открывшемся окне заполните поля     
    • Наименование (отображаемое имя перехода – так будет называться кнопка для смены состояния в карточке объекта).
    • Внутреннее имя - имя перехода в системе
    • Тип перехода («MANUAL» – переход осуществляется по нажатию кнопки, «AUTOMATIC» - переход осуществляется автоматически при наступлении некоего события в системе).
    • Условие перехода (если тип перехода «MANUAL», то это условие для видимости соответствующей кнопки, если тип перехода «AUTOMATIC», то это условие для осуществления самого перехода);
    • Стратегия перехода. Актуально для «Условия перехода» «MANUAL». К переходу может быть привязано несколько согласующих (тех, кто должен нажать кнопку): это могут быть пользователи или группы доступа. Если их несколько, то используется стратегия вычисления условия, когда этот переход нужно выполнить, в соответствии с этим и необходимо установить значение поля «Стратегия перехода»:
      • All - если все согласующие нажали на кнопку перехода;
      • Any - если хотя бы один из согласующих нажал кнопку (фактически это всегда будет первый нажавший ее);
      • Majority - если большинство согласующих нажали на кнопку перехода.
  • Нажмите Создать.
