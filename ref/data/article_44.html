Для добавления в систему нового workflow выполните следующие действия:

  • Создайте новый объект workflow
  • Создайте или отредактируйте схему workflow
  •  Укажите шаг по умолчанию для созданного workflow

В данной статье описаны подробные инструкции по выполнению данных действий.

Добавление объекта workflow

Перейдите к пункту меню Настройки - Жизненные циклы

В открывшемся окне нажмите кнопку  +Добавить запись.

В открывшемся окне заполните поля (обязательные подсвечиваются красным цветом):

  • Имя - название жизненного цикла).
  • Шаг по умолчанию - Шаг workflow, который будет первым для объекта (если шагов пока нет, то поле заполнять не нужно)

                      

Нажмите кнопку Создать.

Редактирование схемы workflow

В объекте Workflow, схему которого нужно отредактировать, нажмите кнопку Редактировать workflow

В открывшемся редакторе схемы посредством drag&drop необходимо выполнить следующие действия:

  • Создать шаги workflow
  • Создать переходы между шагами workflow
  • Обязательно сохранить изменения

Создание шагов workflow

Для этого перетащите иконку состояния Active State из правой панели шаблонов состояний в область редактирования workflow.                                                                     

                                                                                                                                                                                                                                                                                                                                                                                                                     

В открывшемся окне заполните поля (обязательные подсвечиваются красным):

  • Имя (отображаемое имя состояния);
  • Код шага (системное имя состояния).
  • Sort order - определяет порядок кнопки перехода на форме

Нажмите кнопку Создать.

Аналогичным образом создайте все остальные шаги из создаваемого жизненного цикла.

Создание переходов между шагами workflow

Нажмите левой кнопкой мыши в зону красного круга в правой области начального шага ,и не отпуская курсов, проведите его в зону зеленого круга в левой области конечного шага:

     

В открывшемся окне заполните поля:

  • Наименование (отображаемое имя перехода – так будет называться кнопка для смены состояния в карточке объекта).
  • Внутреннее имя - имя перехода в системе
  • Тип перехода («MANUAL» – переход осуществляется по нажатию кнопки, «AUTOMATIC» - переход осуществляется автоматически при наступлении некоего события в системе).
  • Условие перехода (если тип перехода «MANUAL», то это условие для видимости соответствующей кнопки, если тип перехода «AUTOMATIC», то это условие для осуществления самого перехода);
  • Стратегия перехода. Актуально для «Условия перехода» «MANUAL». К переходу может быть привязано несколько согласующих (тех, кто должен нажать кнопку): это могут быть пользователи или группы доступа. Если их несколько, то используется стратегия вычисления условия, когда этот переход нужно выполнить, в соответствии с этим и необходимо установить значение поля «Стратегия перехода»:
    • All - если все согласующие нажали на кнопку перехода;
    • Any - если хотя бы один из согласующих нажал кнопку (фактически это всегда будет первый нажавший ее);
    • Majority - если большинство согласующих нажали на кнопку перехода.


Нажмите Создать.

Аналогичным образом создайте все остальные переходы между шагами  workflow.

Сохранение схемы workflow

После создания всех шагов workflow и переходов между ними обязательно  нажмите на кнопку  Сохранить схему   .

Если до этого вы не указали Шаг по умолчанию workflow, то вернитесь к объекту Worklfow и установите его выбрав из шагов, присутствующих на схеме workflow.
