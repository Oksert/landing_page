Оповещения в системе можно разделить на несколько типов, в соответствии с типом событий, по которым они происходят:

  • Смена статуса жизненного цикла (подробнее описано в разделе Добавление и редактирование оповещений по переходу workflow);
  • Создание объекта
  • Обновление полей объекта
  • Удаление записи

Cоздание нового оповещения по событию создания/изменения и удаления объектов

Перейдите к пункту меню Настройка  –> Оповещения –> Настройка оповещений и в открывшемся окне нажмите кнопку Добавить запись

В открывшемся окне заполните обязательные поля:

Имя - название оповещения

Код сущности - объект системы, к которому будет привязан workflow, шаг которого будет вызывать оповещение

Код события -событие, по которому будет отправлено оповещение.
Возможны следующие события:

  • «2-После добавления» - для отправки оповещения при добавлении объектов
  • «6-После удаления» - для отправки оповещений при удалении объектов
  •  «4-После обновления» - для отправки оповещений при изменении полей объектов

Условие срабатывания - выражение на языке Java, которое преобразуется к булевой переменной, когда оно true – оповещение будет рассылаться. Для использования доступно использование данных о текущей записи record, версии хранящейся в БД записи oldrecord, также  методы классов серверного  и пользовательского API.
Например, для проверки на изменения поля типа «Целое число (Integer)» EXPORTRESULTID условие срабатывания будет таким: 

!CommonUtils.isSame(record.getAsInteger("EXPORTRESULTID"),oldrecord.getAsInteger("EXPORTRESULTID"))

Шаблон оповещения - шаблон оповещения, по которому будет составляться текст оповещения; для того, чтобы добавить новое нажмите на кнопку  +  в поле «Шаблон оповещения». Подробнее о создании шаблонов смотрите в данной статье ниже.

Получатели сообщения - пользователи и группы пользователей, которые получат данное оповещение.  Подробнее о создании получателей оповещения смотрите в данной статье ниже.

Настройка шаблона оповещения

 Для добавления нового шаблона оповещений перейдите к пункту меню Настройка  –> Оповещения –> Шаблон оповещений. 

В открывшемся окне нажмите кнопку Добавить запись и  в открывшемся окне заполните следующие поля:

Имя оповещения - название шаблона оповещения.

Тема - текст темы письма, которое будет отправляться при оповещении.

Текст -текст тела письма, которое будет отправляться при оповещении.
В данном поле поддерживается html-разметка, которую можно настроить как через графический редактора, так и  напрямую изменить HTML-код нажав на кнопку в редакторе. Также можно выполнить настройку в сторонних редакторах (например, этом) и просто вставить результат в виде HTML-кода .

Добавление получателей оповещения

Для добавления получателей оповещения откройте запись оповещения.

Перейдите в секцию «Получатели сообщения», нажмите кнопку Добавить запись, в открывшемся окне заполните следующие поля:

Тип доступа - определяет тип получателя. Возможны следующие варианты:

  • Вычисляемая группа доступа - для отправки оповещения группе доступа, которую необходимо вычислить  на основании объекта или других алгоритмов. Вычисление записывается в поле Вычисление группы/пользователя
  • Вычисляемый пользователь - для отправки оповещения пользователю, которого необходимо вычислить  на основании объекта или других алгоритмов. Вычисление записывается в поле Вычисление группы/пользователя
  • Группа доступа - для отправки оповещения конкретной группе доступа, указанной в поле Код группы/пользователя
  • Пользователь - для отправки оповещения конкретному пользователю, указанному в поле Код группы/пользователя

Код группы/пользователя - имеет смысл при установке типа доступа Группа доступа и Пользователь и должно содержать ID группы/пользователя.

Вычисление группы/пользователя - имеет смысл при установке типа доступа Вычисляемая Группа доступа и Вычисляемый Пользователь и должно содержать расчет соответствующего внутреннего кода на языке Java. Итоговое значение должно быть записано в переменную типа Integer или List recipientObjectId.

Ниже приведен пример данного расчета:

List recipientObjectId = new ArrayList();
List followerList = QueryUtils.getRecordList("SD_FOLLOWERS", "ENTITYTYPEID = 326 and ENTITYID="+record.getKeyValue());
for (EntityDTO follower:followerList)      
recipientObjectId.add(follower.getAsInteger("USERID"));

Исключить текущего пользователя из списка получателей - при необходимости, если пользователь, вызывающий своими действиями данное оповещение, не должен получать его.

После заполнения полей нажмите кнопку Создать, затем Готово

Повторите шаги 4 и 5 для каждого получателя оповещения/группы получателей.
