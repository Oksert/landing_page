Отчет типа «Расписание» показывает ключевые и ресурсы и их занятость различными событиями в календаре.

Для наглядности ресурсы могут быть объединены в группы.

Событие календаря можно создать из любого объекта системы, указав условие создания и связь полей событий и объектов.

Расписание это не только отчет, но и интерактивный инструмент планирования, позволяющие посредством drag&drop создавать и изменять события календаря (а как следствие и объекты системы).

Подробнее о связи между объектами системы и событиями в календаре читайте статью Настройка объекта календарь, связи между событиями календаря и объектами системы.

Для того, чтобы построить отчет типа Расписание необходимо:

- настроить меппинг объектов системы и событий календаря. Подробнее об этом читайте в статье Настройка меппинга объекта и событий календаря.

- создать таблицу типа Запрос с определенным набором полей, которые описаны в данной статье ниже.

Расписание  может использоваться как отдельный отчет, быть включенным в состав дашбордов и отображаться на форме одной записи.

Как и для других отчетов для фильтрации данных Расписания можно использовать поля поиска.

В данной статье описаны шаги по созданию и изменению настроек отчета типа "Расписание".

Создание Таблицы для отчета типа "Расписание"

Перейдите к пункту меню Настройка -> Описание таблиц

На открывшейся странице нажмите кнопку +Добавить запись.

В открывшейся форме заполните обязательные поля:                                                                                                                                                                                        

Название - название таблицы

Системное имя - имя объекта на латинице, которое будет использоваться в бизнес-логике                                                                                                                             

Тип источника данных - укажите значение Запрос

Sql Query - запрос к базе данных на языке SQL, который обязательно содержит следующие поля: 

  • GROUP1 - первый уровень групп ресурсов. Должно содержать поле типа Строка.
  • ...
  • GROUPN - N-ый уровень групп ресурсов. Должно содержать поле типа Строка.
  • EVENTID - ID объекта event, который относится к ресурсу GROUPN.

 Таким образом, каждая строка запроса добавляет занятость определенного ресурса определенным событием.

Пример запроса:

select case when lower(e.event_type) in ('holiday','exception') 
       then '@for_all_group@' 
       else u.full_name end group1,e.calendarevent_id eventid 
from system.ipcalendarevent e
left join v_resource_demand r on (e.entitytype_id = 2234 and e.entity_id = r.resource_demandid)
left join v_personal_activity pa on (e.entity_id = pa.personal_activityid and e.entitytype_id = 2235)
left join v_resource_pool_user ru on  ((ru.user = r.resource) or (pa.user = ru.user))
left join system.ipuser u on (u.user_id = ru.user)
where ((not ru.user  is null) and ((@p_resourcepool is null) or (ru.resourcepool = @p_resourcepool))) 
      or (lower(e.event_type) in ('exception','holiday'))

Где показывается занятость сотрудников подразделения (номер подразделения хранится в поле поиска  @p_resourcepool) работами на проектах, личные активности сотрудников (обучение, отпуска, больничные и т.д.), а также праздники и другие нерабочие дни.

После заполнения полей нажмите кнопку Создать

Добавьте поля таблицы в соответствии с колонками результатов запроса:

После создания всех полей установите галку Активно  и нажмите на кнопку  Cохранить для того, чтобы активировать Таблицу.

Создание отчета типа "Расписание" (SCHEDULER) 

Для создания отчета типа "Расписание" (SCHEDULER) перейдите к пункту меню Настройка->Настройка отображения.

Нажмите на кнопку + Добавить запись и в открывшемся окне заполните поля  следующим образом:

Название - название отчета

Таблица - таблица данные которой нужно отобразить. Укажите таблицу, которая была создана в первом пункте данной статьи

Системное имя - наименование на латинице, которое будет использовано в качестве ссылки на отчет. Таким образом ссылка на создаваемый отчет будет иметь вид: http://:/#/<Системное имя>
где   - имя или ip сервера, на котором установлен web application server системы
          - порт, на котором установлен web application server системы
Например, для отчета с системным именем "my_tasks" ссылка будет иметь вид:  http://runidea.online/#/my_tasks

Тип отображения - для отображения типа "Графика" необходимо указать тип SCHEDULER

Запрос для выборки данных - условие на языке SQL к таблице указанной в поле Таблица, которому должны удовлетворять все записи, попадающие в отчет.  Важно отметить, что к полям объекта нужно обращаться через их системное имя, также как в API системы. Также в запросе можно использовать переменные среды и значения полей поиска текущего отчета, для этого перед полем поиска нужно поставить символ @.

Пример запроса:

assignedto = @currentUserid and workflowstepname = 'inwork' and type = '@p_type'


В данном примере используются следующие переменные:

  • @currentUserid - переменная среды, которая хранит id текущего пользователя системы
  •  @p_type - это ссылка на поле Поиска данного отчета. О том как создать поле поиска и задать ему значение по умолчанию читайте в статье Настройка полей поиска.

Шаблон заголовка - укажите текст, который будет отображаться в заголовке отчета. В заголовке отчета могут быть использованы ссылки на значения полей поиска. Для этого системное имя поля поиска нужно взять в двойные квадратные скобки [[имя поля поиска]].
Например, если отчет отображает задачи определенного типа, значения которого указано в поле поиска p_type, то заголовок будет выглядеть следующим образом:

Задачи типа [[p_type]]

Календарь- укажите календарь (ссылка на объект Календарь системы ), в котором необходимо отобразить события данного отчета. Влияет на отображения выходных и рабочих дней, а также времени рабочего дня

Вычисление календаря - при необходимости можно написать код на языке Java, результатом которого должно стать вычисление ID объекта календарь, в котором нужно отобразить события отчета

События календаря - события, которые можно создавать и изменять в данном отчете. 

Для корректной работы отчета типы события, события, которые отображаются в календаре должны быть включены список разрешенных событий для данного календаря. Также типы событий, указанные в поле События календаря должны быть включены в список разрешенных событий в календаре.

После заполнения поле нажмите на кнопку Сохранить
