Права доступа на переходы между шагами workflow можно дать как группе пользователей, так и одному пользователю, установив при этом одну их стратегий перехода:

  • ANY - переход будет осуществлен если любой из указанных пользователей и групп выполнит переход
  • ALL -  переход будет осуществлен только когда все указанные пользователи и группы выполнят переход

Пользователи и группы могут быть указаны как непосредственно, так и быть вычислены на основании объекта, к которому привязана workflow и других факторов.

Решение по переходу внутри группы может также выполнено на основании стратегии ANY и ALL.

Создание прав доступа на переходы между шагами workflow

Перейди в пункт меню Настройки -> Жизненные циклы

Найдите нужную вам запись, кликните 2 раза и в открывшейся форме нажмите на кнопку Редактировать workflow

В открывшейся схеме workflow дважды кликните по переходу, права доступа к которому нужно указать права.

В открывшейся записи перехода workflow откройте секцию «Права доступа» и нажмите кнопку  + Добавить запись.

В открывшемся окне заполните следующие поля:                                                                                                                                                                                                                                                                                                                    

Тип доступа - определяет кому дает доступ данная настройка. 
Возможные значения:

  • Группа доступа. Доступ предоставляется конкретной группе доступа, которая указана в поле Id группы(пользователя) ;
  • Пользователь.  Доступ предоставляется конкретному пользователю, который указан в поле Id группы(пользователя) ;
  • Вычисляемая группа доступа. Доступ предоставляется  группе доступа, которая вычисляется в поле Вычисление посредством скрипта на Java;
  • Вычисляемый пользователь. Доступ предоставляется  пользователю, который вычисляется в поле Вычисление посредством скрипта на Java.

Стратегия согласования (для групп) показывает количество пользователей в группе необходимое и достаточное для осуществления перехода:

  • ANY (хотя бы один участник группы).
  • ALL (все участники группы).
  • MAJORITY (большинство участников группы).

Id группы/пользователя - содержит id группы или пользователя, которому необходимо предоставить права на осуществление перехода

Вычисление  - содержит расчет соответствующего ID группы доступа или пользователя на языке Java, итоговое значение должно быть записано в переменную accessObjectId типа «Integer» или «List.

Пример вычисления группы доступа:

Integer accessObjectId = 2;
EntityDTO role = QueryUtils.getRecord("ROLE","DISPLAYNAME='Руководитель РСП'");
if (role!=null){
     if (record.get("ORGUNIT")!=null){
             EntityDTO group = QueryUtils.getRecord("ACCESSGROUP","ORGUNIT = "+record.getAsString("ORGUNIT")+" and ROLE="+role.get("ROLEID"));
             if (group!=null)
                accessObjectId  = group.getAsInteger("ACCESSGROUPID");
    }
}

После заполнения всех полей нажмите кнопку  Создать, затем Готово 

Повторите шаг по настройке доступа необходимое количество раз - чтобы все пользователи или группы доступа, которые должны иметь доступ к переводу жизненного цикла, имели соответствующие права.
