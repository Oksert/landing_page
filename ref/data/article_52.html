Отчеты типа  «Круговая диаграмма», «Кольцевая диаграмма» и «Столбчатая диаграмма» позволяют анализировать данные системы отображая их в агрегированном графическом виде.

Как и для других типов отчетов диаграммы можно одинаковым образом строить как на основании таблиц типа Таблица (то есть физических таблиц), так и таблиц типа Запрос (то есть данных, полученных в результате SQL-запроса).

Табличные отчеты могут использоваться как отдельные отчеты, быть включены в состав дашбордов и отображаться на форме одной записи. Все диаграммы могут быть выгружены из системы в форматах PNG и PDF.

Как и для других отчетов для фильтрации данных диаграмм можно использовать поля поиска.

В данной статье описаны шаги по созданию и изменению настроек диаграмм.

Отчет типа «Круговая диаграмма» (PIE CHART)

Отчет типа «Круговая диаграмма» (PIE_CHART) отображает данные в виде круговой диаграммы. Пример отчета типа "Круговая диаграмма" продемонстрирован ниже:

Для создания отчета типа "Круговая диаграмма" (PIE CHART) перейдите к пункту меню Настройка->Настройка отображения.

Нажмите на кнопку + Добавить запись и в открывшемся окне заполните поля  следующим образом:

Название - название отчета

Таблица - таблица данные которой нужно отобразить в виде Круговой диаграммы

Системное имя - наименование на латинице, которое будет использовано в качестве ссылки на отчет. Таким образом ссылка на создаваемый отчет будет иметь вид: http://:/#/<Системное имя>
где   - имя или ip сервера, на котором установлен web application server системы
          - порт, на котором установлен web application server системы
Например, для отчета с системным именем "my_tasks" ссылка будет иметь вид:  http://runidea.online/#/my_tasks

Тип отображения - для отображения типа "Круговая диаграмма" необходимо указать тип PIE_CHART

Запрос для выборки данных - условие на языке SQL к таблице указанной в поле Таблица, которому должны удовлетворять все записи, попадающие в отчет.  Важно отметить, что к полям объекта нужно обращаться через их системное имя, также как в API системы. Также в запросе можно использовать переменные среды и значения полей поиска текущего отчета, для этого перед полем поиска нужно поставить символ @.

Пример запроса:

assignedto = @currentUserid and workflowstepname = 'inwork' and type = '@p_type'


В данном примере используются следующие переменные:

  • @currentUserid - переменная среды, которая хранит id текущего пользователя системы
  •  @p_type - это ссылка на поле Поиска данного отчета. О том как создать поле поиска и задать ему значение по умолчанию читайте в статье Настройка полей поиска.

Шаблон заголовка - укажите текст, который будет отображаться в заголовке отчета. В заголовке отчета могут быть использованы ссылки на значения полей поиска. Для этого системное имя поля поиска нужно взять в двойные квадратные скобки [[имя поля поиска]].
Например, если отчет отображает задачи определенного типа, значения которого указано в поле поиска p_type, то заголовок будет выглядеть следующим образом:

Задачи типа [[p_type]]

Группировать по полю укажите поле, по которому необходимо сгруппировать данные таблицы. Наименования групп станут наименованиями секторов Круговой диаграммы.

Аргумент агрегирующей функции - это поле значения которого будут использованы для расчета агрегатного значения внутри каждой из групп (секторов)

Агрегирующая функция  - функция, которая будет применена к полю, указанному в поле Аргумент агрегирующей функции.

Возможные значения: COUNT,  SUM, AVG, MAX, MIN.

Способ фильтрации -  необходимо указать значение "Форма поиска". После чего у отчета появляется форма поиска, в которой содержатся поля поиска данного отчета. Подробнее о полях поиска читайте в статье Настройка полей поиска.

Интервал обновления, сек - интервал автоматического обновления данных отчета. Значению по умолчанию равно 120 сек. Для того чтобы отчет не обновлялся укажите значение 0.

После заполнения поле нажмите на кнопку Сохранить

Отчет типа "Кольцевая диаграмма" (DONUT_CHART)

Отчет типа «Кольцевая диаграмма» представляет собой данные в графическом виде кольцевой диаграммы. В дополнении к круговой диаграмме отображается суммарное значение по всем секторам диаграммы с кратким описание выводимых данных:

Процесс настройки отчета типа "Кольцевая диаграмма" аналогичен процессу настройки отчета типа "Круговая диаграмма" (описан в данной статье выше) за исключением заполнения следующих полей:

Тип отображения - для отображения типа "Кольцевая диаграмма" необходимо указать тип DONUT_CHART

Поле описания - текст, который будет отображаться в центре кольцевой диаграммы рядом с агрегатным значением.
