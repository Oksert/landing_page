Отчет типа «График» (MULTIPLOT) представляет собой графический отчет, позволяющие анализировать изменение данных по заданным измерениям.

На отчете типа "График" (MULTIPLOT) можно отобразить следующие виды графиков:

  • График - набор точек соединенных между собой
  • Область - набор точек соединенных между собой и, вместе с осями, образующая замкнутое пространство.
  • Столбчатая диаграмма - набор столбцов, отображающих заданные значения через высоту.

На одном графике можно отображать различные серии данных, которые будут различаться по цвету. Также есть возможность объединения нескольких серий, отображаемых в виде столбчатых диаграмм, в группу, которая отображаются в виде одного столбца в одной точке по оси X:

Для того, чтобы построить отчет типа графика необходимо иметь  таблицу типа Таблица (то есть физических таблиц) или таблицу типа Запрос (то есть данных, полученных в результате SQL-запроса) с определенным набором полей, которые описаны в данной статье ниже.

Графики могут использоваться как отдельные отчеты, быть включены в состав дашбордов и отображаться на форме одной записи. Все диаграммы могут быть выгружены из системы в форматах PNG и PDF.

Как и для других отчетов для фильтрации данных диаграмм можно использовать поля поиска.

В данной статье описаны шаги по созданию и изменению настроек графика.

Создание Таблицы для отчета типа «График»

Перейдите к пункту меню Настройка -> Описание таблиц

На открывшейся странице нажмите кнопку +Добавить запись.

В открывшейся форме заполните обязательные поля:                                                                                                                                                                                       

Название - название таблицы

Системное имя - имя объекта на латинице, которое будет использоваться в бизнес-логике                                                                                                                             

Тип источника данных - укажите значение Запрос

Sql Query - запрос к базе данных на языке SQL, который обязательно содержит следующие поля: 

  • XVALUE - значение по оси X графика. Обязательно должно иметь числовой тип данных: Целое число, Вещественное число, Дата и Время, Дата, Время
  • YVALUE - значение по оси Y графика.  Обязательно должно иметь числовой тип данных, как правило это Целое или Вещественное число
  • SERIES - наименование серии. Точки с одной серии будут отображаться одним цветом и, если тип графика "График", то будут последовательно соединены линией определенного цвета
  • GRAPH_TYPE - тип графика. Возможные значения: line - для графика, bar - для столбчатой диаграмма,  area - для области
  • GROUP_NUMBER - наименование группы. Точки, принадлежащие графику типа "Столбчатая диаграмма", входящие в одну группу будут отображены как столбики, стоящие друг на друге
  • Y_AXIS - номер оси по которой указано значение по оси Y. Возможные значения 1 и 2.

Пример запроса:

select 0 as xvalue, -0.11 as yvalue, 'stream0' as series, 'area' as graph_type, 1 as y_axis, cast(null as integer) as group_number

После заполнения полей нажмите кнопку Создать

Добавьте поля таблицы в соответствии с колонками результатов запроса:

В дальнейшем, название поля xvalue будет использовано как название оси X графика, а название поля y_axis как название оси Y графика.

После создания всех полей установите галку Активно  и нажмите на кнопку  Cохранить для того, чтобы активировать Таблицу.

Создание отчета типа График (MULTIPLOT)

Для создания отчета типа "График" (MULTIPLOT) перейдите к пункту меню Настройка->Настройка отображения.

Нажмите на кнопку + Добавить запись и в открывшемся окне заполните поля  следующим образом:

Название - название отчета

Таблица - таблица данные которой нужно отобразить. Укажите таблицу, которая была создана в первом пункте данной статьи

Системное имя - наименование на латинице, которое будет использовано в качестве ссылки на отчет. Таким образом ссылка на создаваемый отчет будет иметь вид: http://:/#/<Системное имя>
где   - имя или ip сервера, на котором установлен web application server системы
          - порт, на котором установлен web application server системы
Например, для отчета с системным именем "my_tasks" ссылка будет иметь вид:  http://runidea.online/#/my_tasks

Тип отображения - для отображения типа "Графика" необходимо указать тип MULTI_PLOT

Запрос для выборки данных - условие на языке SQL к таблице указанной в поле Таблица, которому должны удовлетворять все записи, попадающие в отчет.  Важно отметить, что к полям объекта нужно обращаться через их системное имя, также как в API системы. Также в запросе можно использовать переменные среды и значения полей поиска текущего отчета, для этого перед полем поиска нужно поставить символ @.

Пример запроса:

assignedto = @currentUserid and workflowstepname = 'inwork' and type = '@p_type'


В данном примере используются следующие переменные:

  • @currentUserid - переменная среды, которая хранит id текущего пользователя системы
  •  @p_type - это ссылка на поле Поиска данного отчета. О том как создать поле поиска и задать ему значение по умолчанию читайте в статье Настройка полей поиска.

Шаблон заголовка - укажите текст, который будет отображаться в заголовке отчета. В заголовке отчета могут быть использованы ссылки на значения полей поиска. Для этого системное имя поля поиска нужно взять в двойные квадратные скобки [[имя поля поиска]].
Например, если отчет отображает задачи определенного типа, значения которого указано в поле поиска p_type, то заголовок будет выглядеть следующим образом:

Задачи типа [[p_type]]

Имя серии - укажите поле Таблицы, в котором хранятся серии точек (в примере выше это поле series)

Значение по X - укажите поле Таблицы, в котором хранятся координаты точек по оси  X (в примере выше это поле xvalues)

Значение по Y - укажите поле Таблицы, в котором хранятся координаты точек по оси  Y (в примере выше это поле yvalues)

Поле с типом графика - укажите поле Таблицы, в котором хранятся типы графиков (в примере выше это поле graph_type)

Номер группы - укажите поле Таблицы, в котором хранятся группы точек (в примере выше это поле group_number)

Поле с номером оси Y - укажите поле Таблицы, в котором хранится номер оси Y (в примере выше это поле y_axis )

После заполнения всех полей нажмите на кнопку Cоздать
