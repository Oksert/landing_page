При создании одного объекта из другого можно настроить перенос полей из текущего объекта в создаваемый.

Для решения этой задачи используется объект настройки Меппинг полей.

Пусть объект Функциональное требование имеет виртуальное поле Задачи, которое хранит в себе все записи из объекта задачи, относящиеся к Проекту. Тогда, если установить связи полей между Проект и Задача со следующим вычислением:

destination.set("REQUIREMENTID", source.get("SD_REQUIREMENTID"));

то при создании записи объекта Задача из формы одной записи Проекта, то поля со внутренним именем SD_REQUIREMENTID объекта Задача проставится в соответствии с полем Проекта REQUIREMENTID, из которого создавалась запись

       

Создания нового mapping полей

  1. Перейдите к пункту «Система управления» панели навигации, расположенной в левой части экрана. Выберите пункт «Меппинг полей».
  2. На открывшейся странице нажмите кнопку +Добавить запись.
  3. В открывшейся форме укажите следующие поля:                                                                                                                                                                                                                                                                                                                     
    • Source Entitytype Id - таблица-источник данных.
    • Source Field Name - поле из таблицы-источника данных, по которому запись из нее связывается с таблицей-получателем данных.
    • Destination Entitytype Id - таблица-получатель данных.
    • Destination Field Name  - поле из таблицы-получателя данных, по которому запись из нее связывается с таблицей-источником данных.
    • Mapping Calculation - расчет данных для передачи из таблицы-источника в таблицу-получатель.

Пример заполнения поля «Mapping Calculation»:

destination.set("FIELD_NAME_DEST",source.getAsInteger("FIELD_NAME_SRC"));
  • «destination» - запись из таблицы «Destination Table Name»,
  • «source» - запись из таблицы «Source Table Name»,
  •  «FIELD_NAME_DEST» – поле таблицы «Destination Table Name»
  • «FIELD_NAME_SRC» из таблицы «Source Table Name».

В приведенном примере тип обоих полей – Integer, в случае другого типа данных необходимо использовать соответствующий ему метод (например, «getAsString» для строк). При этом можно перенести данные по нескольким полям – для этого укажите правила переноса данных для каждого такого поля.

4. После внесения всех данных нажмите на кнопку Создать.

Редактирование существующего mapping полей

  1. Перейдите к пункту «Настройки» панели навигации, расположенной в левой части экрана. Выберите пункт «Меппинг полей».
  2. На открывшейся странице в нижней части экрана представлен список всех записей о Mapping полей системы. Найдите необходимую (при необходимости используйте поиск) и двойным кликом откройте запись.
  3. В открывшейся форме измените необходимые поля:
    • Source Entitytype Id - таблица-источник данных.
    • Source Field Name - поле из таблицы-источника данных, по которому запись из нее связывается с таблицей-получателем данных.
    • Destination Entitytype Id - таблица-получатель данных.
    • Destination Field Name  - поле из таблицы-получателя данных, по которому запись из нее связывается с таблицей-источником данных.
    • Mapping Calculation - расчет данных для передачи из таблицы-источника в таблицу-получатель.
  4. После внесения всех изменений нажмите на кнопку Сохранить
