Календарь - это инструмент системы, который позволяет:

  • Задать рабочие графики
  • Настроить исключения и праздники
  • Задать расчет дат и длительности операций согласно рабочему календарю
  • Отобразить объекты системы в календарном представлении или в виде расписания
  • Планировать загрузку ресурсов на основании в интерактивных инструментах планирования (Расписание и Диаграмма Ганта)

В данной статье описаны шаги по настройке и использованию календаря и его событий.

Календарь и события календаря

Cоздание календаря 

Для создания нового календаря выполните следующие действия:

1. Перейдите к пункту меню Настройка –> Календари -> Настройка календарей

2. На открывшейся странице нажмите кнопку +Добавить запись.               

3. В открывшейся форме укажите следующие поля:

  • Имя - укажите название календаря
  • Разрешенные типы событий - выберите не менее одного типа событий, которые будут доступны для использования в календаре

4. Нажмите кнопку Создать

5. Перейдите в секцию «Рабочий график»

6. Укажите каждому дню недели время начала и окончания работы, а так же его тип: рабочий (WORKING) или не рабочий(NON-WORKING), для этого двойным кликом откройте соответствующую строку и внесите необходимые изменения. После внесения правок нажмите кнопку Готово.

Стандартные типы событий: Праздник и Исключение

По умолчанию любой календарь может иметь два типа событий: Праздник (HOLIDAY) и Исключение (EXCEPTION).

Тип события Праздник (HOLIDAY) позволяет внести в рабочий календарь все исключительные ситуации, например, государственные праздники. Событие типа "Праздник" всегда является нерабочим днем.

Тип событий Исключение (EXCEPTION) позволяет вносить в график дополнительные исключения, например, рабочий день в субботу. Событие типа "Исключение" может быть как рабочим, так и нерабочим днем.

Для редактирования стандартных событий календаря перейдите на вкладку Праздники и Исключения формы Календаря.
Там можно как создать новое событие, указав: заголовок события, тип события, дата начала и окончания, а также тип дня (WORKING или NON-WORKING), а также отредактировать ранее созданные события.

Создание и редактирование событий календаря

Для того, чтобы создать новое событие календаря необходимо перейти по ссылке: http://:port/#/calendarevent,
где
- доменное имя сервера, на котором установлена Система

- порт на котором установлена система.

В открывшемся списке вы можете либо выбрать событие, которое необходимо отредактировать сделав по нему двойной клик, либо создать новое событие нажав на кнопку  +Добавить запись

Для создания события нужно заполнить следующие поля:

  • Заголовок события
  • Тип события
  • Календарь
  • Начало
  • Окончание
  • Тип дня (WORKING или NON-WORKING)
  • Тип повторения

Также события могут быть созданы и отредактированы через отчет типа Расписание и Диаграмма Ганта.

Расчеты длительности и дат операций с учетом трудового календаря

Созданные календари с заданными праздниками и исключениями можно использовать для расчета дат операций.

Для данных расчетов в системе реализовано специальное API, которое подробно описано в статье Серверные методы и классы API систем, смотрите описание методов класса CalendarUtils.

Создание событий календаря из объектов системы

Для того, чтобы отображать объекты системы в календарном представлении, а также использовать инструменты интерактивного планирования  (Расписание и Диаграмма Ганта) нужно настроить автоматическое создание событий календаря на основании данных объекта.

Для этого нужно:

  • Создать тип события календаря, который будет соответствовать объекту
  • Настроить меппинг объекта  и события календаря, указав: соответствие полей события и объекта, условие создания события, календарь события и другие настройки
  • При необходимости настроить гибкие расчеты при синхронизации объектов и событий

Ниже описано как выполнить данные пункты.

Создание типа события календаря

Перейдите к пункту меню Настройка –> Календари -> Типы событий

 На открывшейся странице нажмите кнопку +Добавить запись  и заполните  поля:

  • Название  - название события для отображения пользователем
  • Внутреннее имя  - наименование типы события на латинице для использования в настройках бизнес-логики системы

Настройка меппинга между объектами и событиями календаря

Перейдите к пункту меню Настройка –> Календари -> Связь календарей и объектов

В открывшемся списке нажмите кнопку +Добавить запись и укажите следующие поля:                                                                                                                                                                                                                                                                                                                                                                        

 
  • Entitype Id - имя объекта на основании которого нужно создать событие календаря.
  • Тип события - тип события, который будет соответствовать указанному объекту
  • Day Type - тип дня, которым будет считаться созданное событие. Возможные варианты:  WORKING и NON-WORKING
  • Условие создания события  - условие на языке JAVA, при выполнении которого нужно создавать событие. Если условие выполняется, то событие будет создано, есил условие перестанет выполняться, то событие будет удалено
  • Calendar Id - Id календаря, к которому нужно привязать событие.
  • Calendar Id Calculation - код на Java для вычисления внутреннего номера календаря. В результате должно вернуться значение переменной calendarId. 
  • Поле с именем события - пол объекта, из которого нужно скопировать имя события.
  • Поле с датой начала события - поле объекта, из которого нужно скопировать дату начала события.
  • Поле с датой окончания события - поле объекта, из которого нужно скопировать дату окончания события.
  • Цвет - цвет события. Должно содержать значение цвета в формате шестнадцатеричного кода с использованием маски #RRGGBB или один из стандартных HTML-цветов (red, blue, green)
  • Color Code Inactive - цвет события, когда объект становится неактивным (у записи снимается галка Активно). Настроить условие активности записи можно в настройках Таблицы, читайте об этом в статье Добавление и редактирование таблиц
  • Разрешить изменение дат на клиенте - определяет возможность редактирования дат через интерактивные инструменты планирования (Расписание и Диаграмма Ганта) 
  • Dragndrop Condidition - условие на языке Java, возвращающее true или false. Определяет возможность изменения планового начала и окончания события посредством перетаскивания события в отчете типа Расписание 
  • Условие изменения длительности - условие на языке Java, возвращающее true или false. Определяет возможность пользователя изменять длительность события посредством сужения/расширения события в отчете типа Расписание 

После заполнения полей нажмите кнопку Создать.

После настройки меппинга между объектами и событиями календаря события будут автоматически создаваться при создании объектов и выполнений условий. 

При изменений объектов атрибуты события календаря будут автоматически синхронизироваться с атрибутами объекта. Если после обновления объекта условие, указанное в меппинге перестанет выполняться, то событие будет удалено.

Интерактивные инструменты планирования также подразумевают создание и изменение событий, а, как следствие, и событий календаря. В этом случае атрибуты объекта будут изменяться согласно атрибутам события и настроенного меппинга.

Настройка гибких расчетов при синхронизации объектов и связанных событий

Для выполнения гибких расчетов при синхронизации объектов и событий календаря в меппинге есть возможность описать расчеты на языке Java, используя обращения к текущему объекту и событию.

Доступны следующие поля:

Record2Event Mapping Calculation - код, который выполняется при создании/изменении объекта и обновления события календаря. В коде можно использовать следующие переменные: 

  • record - текущий объект
  • event -  текущее событие

Объект event - это экземпляр класса  EntityDTO, который имеет следующие атрибуты:

  • displayname - наименование события
  • starttime . - дата начала события
  • endtime - дата окончания события
  • calendarid - календарь события
  • colorcode - цвет события
  • entitytypeid - код таблицы связанного объекта
  • entityid - код объекта
  • parentCalendareventId - родительское событие (имеет смысла при настройке редактирования через  Диаграмма Ганта)
  • PREDECESSORS - массив событий - предшественников (имеет смысла при настройке редактирования через  Диаграмма Ганта)
  • EVENTLAG - массив чисел - отступлений предшественников от текущего события  (имеет смысла при настройке редактирования через  Диаграмма Ганта)

Для получения значений объекта можно воспользоваться методом event.get("<название атрибута>").

Для того, чтобы изменить значение можно воспользоваться методом event.set("<название атрибута>").

Ниже представлен пример расчет наименования события на основе данных объекта:

if (record.get("resourcepool")!=null){
    EntityDTO team = new EntityDTO("resource_pool",record.getAsInteger("resourcepool"));
    if (team!=null)
        event.set("DISPLAYNAME", team.get("fullname"));
}

Event2Record Mapping Calculation - код, который выполняется при создании/изменении события и обновления объекта (например, через отчет типа  Расписание ). В коде можно использовать следующие переменные: 

  • record - текущий объект
  • event -   текущее событие
  • oldevent - текущее событие до внесения изменений

Также для того, чтобы из отчет типа Расписание передать наименование групп (также как это выглядит в отчете) в создаваемую из этого отчета объект, нужно использовать переменные: 

  • ContextUtils.getEnvironmentVars().get("group_1") - для групп
  • ContextUtils.getEnvironmentVars().get("group_2") - для подгрупп
  • ContextUtils.getEnvironmentVars().get("group_3") - для под-подгрупп ит.д.

Ниже представлен пример расчета сотрудника, на которого создается запрос через отчет типа Расписание:

if (ContextUtils.getEnvironmentVars().get("group_1")!=null){
    log.info("группа найдена - "+ContextUtils.getEnvironmentVars().get("group_1"));
    EntityDTO usr = QueryUtils.getRecord("USER","FULLNAME=\""+ContextUtils.getEnvironmentVars().get("group_1")+"\"");
    if (usr!=null)
        record.set("resource",usr.getKeyValue());
}

API для "ручной" синхронизации объектов системы и событий календаря

После того, как настроен меппинг объектов и событий календаря синхронизация начинает работать автоматически.

Для того, чтобы вручную провести синхронизацию объекта и события можно использовать API (например, для того чтобы обработать исторические данные):

  • CalendarUtils.syncEntityWithCalendar(record) - добавляет/обновляет событие календаря. В случае, если условие не выполняется - событие календаря автоматически удаляется.
  • CalendarUtils.removeEntityFromCalendar(record); - удаляет событие календаря.
