Настройте выполнение оповещений по созданному переходу, для этого:

  1. Откройте карточку перехода, оповещение по которому необходимо производить.
  2. Перейдите в секцию «Оповещения», нажмите кнопку «Добавить запись».
  3. В открывшемся окне заполните обязательные поля:
    • Имя (название оповещения);
    • Код сущности (объект системы, к которому будет привязан workflow, шаг которого будет вызывать оповещение);
    • Условие срабатывания (выражение на языке Java, преобразующееся к булевой переменной, когда оно возвращает значение «true» – оповещение будет рассылаться);
    • Шаблон оповещения (Шаблон оповещения, по которому будет составляться текст оповещения, нажмите на кнопку «+» в поле «Шаблон оповещения»;
      • в открывшемся окне заполните следующие поля:
        • Имя оповещения (название шаблона оповещения).
        • Тема (Текст темы письма, которое будет отправляться при оповещении).
        • Текст (Текст тела письма, которое будет отправляться при оповещении).
      • нажмите кнопку «Создать», затем «Готово».
    • Перейдите в секцию «Получатели сообщения», нажмите кнопку «Добавить запись», в открывшемся окне заполните следующие поля:
    • Внутреннее имя (название получателя оповещения).
    • Тип доступа:
      • вычисляемая группа доступа (если получатель оповещения - группа доступа, внутренний код которой вычисляется в поле «Вычисление группы/пользователя»);
      • вычисляемый пользователь (если получатель оповещения - пользователь, внутренний код которого вычисляется в поле «Вычисление группы/пользователя»);
      • группа доступа (если получатель оповещения - группа доступа, внутренний код которой известен);
      • пользователь (если получатель оповещения - пользователь, внутренний код которого известен).
    • Код группы/пользователя (в случае «типа доступа» «вычисляемая группа доступа» или «вычисляемый пользователь» - любое целое число, например, «9999», иначе – соответствующий внутренний код).
    • Вычисление группы/пользователя (в случае «типа доступа» «вычисляемая группа доступа» или «вычисляемый пользователь» - расчет соответствующего внутреннего кода на языке Java, итоговое значение должно быть записано в переменную типа «Integer» или «List<Integer>» «recipientObjectId»).
    • Исключить текущего пользователя из списка получателей (при необходимости, если пользователь, вызывающий своими действиями данное оповещение, не должен получать его).
  4. Нажмите кнопку «Создать», затем «Готово».
  5. Повторите шаги соответствующие шаги руководства для создания каждого получателя оповещения/группы получателей.
  6. Нажмите кнопку «Создать», затем «Готово».
  7. Повторите процедуру создания оповещения для каждого оповещения на этот переход.
