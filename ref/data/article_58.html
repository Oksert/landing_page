Для создания нового предупреждения выполните следующие действия:

  1. Перейдите к пункту «Настройка» панели навигации, расположенной в левой части экрана. Выберите пункт «Предупреждения».
  2. На открывшейся странице нажмите кнопку «Добавить запись».
  3. В открывшейся форме укажите следующие поля:                                                                                                                                                                                                                                                                                            
  • Наименование (название предупреждения в системе).
  • Уровень исключения («зеленый» - наименьший уровень внимания к предупреждению, «желтый» - средний, «красный» - максимальный).
  • Таблица (системное имя таблицы, по которой будут создаваться предупреждения).
  • Активно (указывает включена/отключена генерация предупреждений).
  • Условие срабатывания (код на языке Java, с использованием серверных методов и классов API системы (подробнее см. раздел Серверные методы и классы API системы) и Пользовательский API системы (Class Library) (подробнее см. раздел Пользовательский API системы (Class Library)), описывающий условия создания предупреждения (для безусловного создания – «true»)).
  • Пример условия срабатывания:
("1-planned".equals(record.getAsString("WORKFLOWSTEPNAME")))  && 
(record.get("PLANNED_START") != null) && 
(record.get("ORGUNIT")!=null) && !Lib.RequestUtils.isFullProcessRequest(record)
  • Расчет времени срабатывания (код на языке Java, с использованием серверных методов и классов API системы (подробнее см. раздел Серверные методы и классы API системы) и Пользовательский API системы (Class Library) (подробнее см. раздел Пользовательский API системы (Class Library)), описывающий расчет времени первого появления предупреждения – рассчитывается один раз – при первом формировании предупреждения).
  • Пример расчета времени срабатывания:
EntityDTOorgUnit = newEntityDTO("R_ORGUNIT",record.getAsInteger("ORGUNIT"));
Object alertTime = null;
if (orgUnit!=null){
        int delta = CommonUtils.nullSub(orgUnit.getAsInteger("HOUR_DIFF"),0);
        alertTime =  new Timestamp(record.getAsDateTime("PLANNED_START").getTime() + (long)((8.5-delta)*60*60*1000));
}
  • Время следующего срабатывания (код на языке Java, с использованием серверных методов и классов API системы (подробнее см. раздел Серверные методы и классы API системы) и Пользовательский API системы (Class Library) (подробнее см. раздел Пользовательский API системы (Class Library)), описывающий расчет времени повторного появления предупреждения – необходимо, если объект взяли «На контроль» - рассчитывается столько раз, сколько объект берут на контроль до тех пор, пора «Условие срабатывания» выполняется).
  • Пример расчета времени следующего срабатывания:
EntityDTO orgUnit = new EntityDTO("R_ORGUNIT",record.getAsInteger("ORGUNIT"));
Object alertTime = null;
if (orgUnit!=null){
        int delta = CommonUtils.nullSub(orgUnit.getAsInteger("HOUR_DIFF"),0);
        java.util.Calendar cal = Calendar.getInstance();
        cal.set(Calendar.HOUR_OF_DAY,8-delta);
        cal.set(Calendar.MINUTE,30);
        cal.set(Calendar.SECOND, 0);
        alertTime = new java.sql.Timestamp(cal.getTime().getTime() + 24*60*60*1000);
}

    4. После внесения всех данных нажмите на кнопку Создать.

    Для редактирования существующего предупреждения выполните следующие действия:

    1. Перейдите к пункту «Настройка» панели навигации, расположенной в левой части экрана. Выберите пункт «Предупреждения».
    2. На открывшейся странице в нижней части экрана представлен список всех предупреждений системы. Найдите необходимое предупреждение (при необходимости используйте поиск) и двойным кликом откройте запись.
    3. В отрывшейся форме измените необходимые поля:
      • Наименование (название предупреждения в системе).
      • Уровень исключения («зеленый» - наименьший уровень внимания к предупреждению, «желтый» - средний, «красный» - максимальный).
      • Таблица (системное имя таблицы, по которой будут создаваться предупреждения).
      • Активно (указывает включена/отключена ли генерация предупреждений).
      • Условие срабатывания (условия, при выполнении которых создается предупреждение).
      • Расчет времени срабатывания (код на языке Java, описывающий расчет времени первого появления предупреждения – рассчитывается один раз – при первом формировании предупреждения).
      • Время следующего срабатывания (код на языке Java, описывающий расчет времени повторного появления предупреждения – необходимо, если объект взяли «На контроль» - рассчитывается столько раз, сколько объект берут на контроль до тех пор, пора «Условие срабатывания» выполняется).
    4. После внесения всех изменений нажмите на кнопку Сохранить.
