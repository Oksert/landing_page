Для создания и редактирования таблиц необходимо иметь права администратора системы.

Создание таблицы

  1. В верхней части экрана появится Меню, перейдите в пункт меню Настройка -> Описание таблиц

                                 

  2. Нажмите кнопку Добавить запись

   3. В открывшемся окне заполните обязательные поля:

  • Название (название объекта для пользователей системы).
  • Системное имя (внутреннее название объекта в системе, не будет видно пользователям).
  • Тип источника данных (Таблица - для создания объекта, Запрос - для формирования отчетов). В данном случае должен быть указан Таблица
  • Базовая таблица – шаблон, который будет использоваться для создания нового объекта. Он влияет на количество и наименование полей различных типов, которые будут созданы и привязаны к создаваемому объекту. Для выбора доступны все активные таблицы, как заранее готовые шаблоны, в том числе:
    • Справочник (Reference)
    • Данные (Data)  

        После заполнения поля Базовая таблица автоматически будут заполнены поля:
        - Кол-во полей типа Integer
       -  Кол-во полей типа Double
       -  Кол-во полей типа Boolean
       -  Кол-во полей типа String
       -  Кол-во полей типа Text
       -  Кол-во полей типа DateTime
       -  Кол-во    полей типа Date, Кол-во полей типа Time.

Эти поля доступны для редактирования:

                                                                                                                                                                                                               

При необходимости также можно указать следующие поля:

  • Аудит Включен - если по каким-либо полям таблицы необходимо вести историю
  • Используется WorkFlow  - если у объекта будет собственный жизненный цикл, нужно установить галку "Используется Workflow" и после создания объекта указать значение по умолчанию с соответствующим внутренним кодом жизненного цикла в поле «WORKFLOWID».
  • Поиск по текстовым полям - включает полнотекстовый поиск по всем текстовым полям и вложениям системы. О том, как установить вес конкретного поля в поиске см. раздел Добавление и редактирование полей таблицы
  • Active condition - в данном поле можно написать условие активности объекта (снимается или ставится галка "Активно" объекта). Данное поле представляет собой выражение на языке Java, которое преобразуюется к булевой переменной. Если возвращаемое значение условия true, то объект активен, если false - неактивен.

    • Пример условия: 

!"completed".equals(record.getAsString("WORKFLOWSTEPNAME"))

  • Абстрактная таблица (если таблица будет служить шаблоном для создания других таблиц, но сама содержать данных не будет).

4. После заполнения полей нажмите кнопку Создать

Активация таблицы

После заполнения обязательных полей и создания таблицы у таблицы генерируются атрибуты, соответствующие указанному при создании шаблонов. Теперь эти атрибуты можно редактировать, удалить, создать новые. Подробнее в статье Добавление и редактирование полей объектов.

При этом физически таблица еще не создана, чтобы ее создать нужно поставить галку Активно и нажать на кнопку Сохранить.
После этого в системе автоматически будут созданы:

  • 2 таблицы в базе данных с именами, которые указаны в полях SQL Имя Базовой Таблицы и SQL Имя Таблицы Атрибутов
  • При Типе источника, отличным от Запрос, автоматически генерируются настройки отображения типов для формы одной записи (тип DETAILS) и форма списка (тип LIST), подробнее о настройках отображения смотрите статью Редактирование отображений

После выполнения данных пунктов можно переходить к Настройке отображения
