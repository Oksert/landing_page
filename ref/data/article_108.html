1.     Как снять дамп:

Из командной сроки сервера выполнить команду:

pg_dumpall -Upostgres -c --file="C:\Dumps\Auto\saas_ip_dev_auto.sql" 

Дамп будет сохранен в файл C:\Dumps\Auto\saas_ip_dev_auto.sql

2.     Как восстановить дамп:

1.Скопировать файл с дампом в папку C:/temp/

2.Зайти в командную строку при помощи команды (при необходимости нужно ввести пароль P@$$w0rd):

psql -U postgres

3. Удалить старую базу при помощи команды (или ту которую нужно):

drop database ip_dev;

4. Создать базу данных ip_dev (именно ip_dev, даже если будем восстанавливать другую базу):

create database ip_dev;

5. Восстановить базу (дождаться окончания выполнения скрипта):

\i c:/temp/saas_ip_dev_auto.sql

6. Если база назвалась не ip_dev, то зайти в PGAdmin III и в интерфейсе переименовать базу, кликнув на название базы правой кнопкой мыши.
